# Stamp Scene Model

This directory contains the Flutter development 3D asset for the stamp screen.
The GLB is generated by `tools/create_stamp_scene_glb.py`, a pure Python
pipeline that analyzes reference images, builds an image-matched procedural
product-style stamp, embeds the decal texture, and writes a Flutter-ready GLB.

## Files

- `stamp_scene.glb`: generated GLB loaded by the Flutter app.
- `stamp_decal_transparent.png`: transparent decal texture embedded into the GLB.

## Flutter Usage

```dart
_modelController.playAnimation(
  animationName: 'Stamp',
  loopCount: 1,
);
```

Recommended timing:

```dart
static const Duration stampDuration = Duration(milliseconds: 1150);
static const Duration impactTiming = Duration(milliseconds: 500);
```

Asset registration:

```yaml
flutter:
  assets:
    - assets/models/stamp_scene.glb
```

## Regeneration

Place at least these input images in the project:

- `assets/reference/stamp_front.png`: front product reference for silhouette, color, and proportions.
- `assets/reference/safety_stamp_decal.png`: stamp mark/decal that appears on the paper and underside plate.

Optional inputs:

- `assets/reference/stamp_views.png`: combined views for color/proportion refinement.
- `assets/reference/stamp_top.png`: top view for hole placement and cap proportions.
- `assets/reference/stamp_bottom.png`: underside view for ink plate proportion hints.

Run from the project root. Blender is not required:

```bash
python tools/create_stamp_scene_glb.py \
  --front assets/reference/stamp_front.png \
  --decal assets/reference/safety_stamp_decal.png \
  --views assets/reference/stamp_views.png \
  --out assets/models/stamp_scene.glb

python tools/inspect_stamp_scene_glb.py assets/models/stamp_scene.glb
```

The script removes near-white pixels from the decal and writes
`assets/models/stamp_decal_transparent.png`, then embeds that PNG into the GLB.
It keeps source reference images unchanged.

## Designer Model Replacement Contract

When replacing this procedural model with a production designer export, keep
these names stable:

- animation clip name: `Stamp`
- root node: `StampRoot`
- paper decal node: `InkDecal`
- paper decal child: `InkDecalPlane`
- bottom plate node: `StampInkPlate`
- paper node: `Paper`
- top camera: `TopDownOrthographicCamera`
- front camera: `FrontPreviewCamera`

The paper ink mark must read normally from top-down view:
`SAFETY STAMP` on the upper arc and `SEOLLEYEON` on the lower arc. The generated
GLB is authored directly in glTF's Y-up coordinate system, so `StampRoot` moves
down toward the paper along Y and rebounds upward. Avoid external absolute
texture paths; textures should be embedded in the GLB.
